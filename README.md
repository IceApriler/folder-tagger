# Folder Tagger

**Folder Tagger** 是一款专为 VS Code 设计的高性能企业级资源文件标记底座扩展。它可以让您及您的团队直观、安全、高效地在庞杂的业务树堆中建立标记追踪和整理体验。不依赖笨重的外部库，纯粹基于本地文件映射及内存缓存。

## 🌟 核心特性

- **可视化徽标挂载 (`File Decoration`)**：
  在系统原生及自定义资源管理器中，为标定的文件和文件夹全自动注入带有特殊徽章 (`T`) 和主题色 (`charts.blue`) 的视觉展示，在千万个代码文件中瞬间点亮你专注的心智目标。

- **多端协作守护屏障 (`FileSystemWatcher`)**：
  团队合作时如果另一名成员提交并更新了底层用来追踪标签的 `.vscode/tags.json` 文件，系统具备敏锐的热加载感知力。它将无需重拉进程即可在底层完成瞬态覆盖热刷新，完全杜绝协作导致的脏状态污染与互扫隐患。

- **极致优化的性能护航**：
  - **懒加载 (`Lazy Load`)**：视图仅依照点击深度和可见区域进行展开按需获取信息。
  - **防黑洞过滤池**：默认硬编码无视并拦截类似 `.git/`、`node_modules/` 以及 `dist/` 的大件隐藏结构避免引发读取崩溃和 OOM。
  - **500ms 重核防抖**：通过内置的安全倒计时锁定，无论瞬时增加、修改或删除多少标签操作，始终保证单线单通道在半秒一次内实现无损写盘保障。

- **自我愈合（位置自追踪机制）**：
  直接使用工作区文件管理器进行无论上千上万份规模代码的更名、深层迁移嵌套还是放入回收站，文件对应的标签绑定永远自动自我追寻定位，即使被删去也能同步实现无残留自动清洗。 

- **完美的原生操作缝合**：
  - 支持快捷多维打标签和 `QuickPick` 滑动精准移除单条 Tag。
  - 单击即可秒切至主编辑器（自带了 `vscode.open` 接管）。
  - 除了针对 Tag 的增删改，依然提供接管了核心系统级别删除至“回收站”、重命名和开启相对终端的原生右键替代命令。


## 🛠 使用方法

1. **观察启动**
   应用激活后点击侧边活动栏的最底部新装入的 `Folder Tagger` 视图面板。
2. **快速打标**
   将鼠标拂过任意不包含在 `屏蔽规则` 下的普通文件夹以及代码实体文件上，点击右侧行内（Inline 浮框内）的 **`+` 加号命令** 开启写入支持，你也可以批量以逗号输入（如 `UI组件, 数据模型, 待确认`）。
3. **撤除标注**
   文件被标记后，使用**单击右键**打开资源菜单，并下调至点击名为 `Remove Tag...` 的操作。如果目标带有一大串标记词群，将引发弹出式的多重选择剥离面板。


## 📝 数据存储流规则
所有针对标签的处理记录将会最终指向您项目的如下文件层级中以相对路径存储并参与 Git 版本同步记录。
```json
.vscode/tags.json
```
